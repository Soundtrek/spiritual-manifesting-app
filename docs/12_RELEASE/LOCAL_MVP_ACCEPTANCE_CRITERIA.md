# Local MVP Acceptance Criteria

## Purpose

Define the minimum local acceptance pass before limited beta use.

## Acceptance Criteria

- Dashboard loads without runtime errors and links to Daily, Intentions, Journal, Reflection, and Settings.
- Daily Alignment saves mood, intention, gratitude, and reflection in this browser after reload.
- Intentions can be created, edited, status-updated, deleted, and retained after reload.
- Journal entries can be created, edited, deleted, filtered, and retained after reload.
- Reflection dashboard displays calm placeholder insights without implying active AI analysis.
- Settings can clear local SMA data after confirmation.
- Mobile and narrow desktop layouts remain readable with no overlapping text or broken controls.
- Copy remains grounded, non-promissory, and clear that outcomes are not guaranteed.
- Known limitations are visible in release docs and not contradicted by app copy.

## Not Required For Local MVP Acceptance

- Authentication.
- Account sync.
- Backend database storage.
- AI-generated reflections.
- Export or import.
- Production deployment readiness.
