# Privacy Principles

## Purpose

The app will handle deeply personal data. Privacy must be part of the design from day one.

## Sensitive Data Types

- Journal entries
- Intentions
- Emotional check-ins
- Mood history
- Gratitude entries
- Ritual history
- AI reflection summaries
- Synchronicity logs

## Principles

### Private by Default

User reflections should be private unless the user explicitly chooses otherwise.

### Minimal Collection

Only collect data needed for the app experience.

### User Ownership

Users should be able to export and delete their own data.

### Clear AI Usage

Users should understand when their content is used to generate AI reflections.

### No Surprise Sharing

No journal or intention content should be shared publicly, socially, or externally without explicit user action.

### No Emotional Ad Targeting

The app must not use sensitive emotional or spiritual data for manipulative targeting.

## Future Requirement

Before beta, create a concrete privacy model covering storage, AI processing, retention, export, deletion, and account closure.
