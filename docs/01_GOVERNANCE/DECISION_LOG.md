# Decision Log

This document records major project decisions, why they were made, and what consequences they create.

## Format

Each decision should use this structure:

```text
Date:
Decision:
Reason:
Alternatives Considered:
Consequence:
Status:
```

---

## Decision 001 — Planning-first repository

Date: 2026-05-24
Decision: The project starts as a planning-first GitHub repo before production code.
Reason: Prevent scope drift and create SAMMA-style continuity from day one.
Alternatives Considered: Start coding immediately with VS Code/Copilot.
Consequence: Slower start, safer foundation.
Status: Active

---

## Decision 002 — Grounded spirituality, not supernatural certainty

Date: 2026-05-24
Decision: The app may use spiritual and manifestation language, but must avoid guaranteed outcomes, supernatural certainty, or coercive framing.
Reason: Protect users and make the product emotionally safe.
Alternatives Considered: Full mystical/oracle-style app.
Consequence: The app becomes more trustworthy and sustainable.
Status: Active
